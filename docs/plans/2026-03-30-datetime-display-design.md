# yls-code 时间显示统一设计

## 目标

修复概览页把 Unix 时间戳裸显示为 `1774851324` 的问题，并统一桌面端所有“展示型日期时间”输出，全部按当前操作系统的时区和 locale 显示。

## 现状

- 日志时间、趋势时间桶来自后端 ISO UTC 字符串
- 同步时间来自 SQLite `strftime('%s', 'now')`，是 Unix 秒级时间戳
- 前端没有统一格式化层，多个页面和图表直接显示原值

## 方案

采用前端统一格式化方案，不修改数据库存储格式：

1. 新增时间工具模块，负责：
   - 识别 Unix 秒、Unix 毫秒、ISO 日期字符串、纯日期字符串
   - 按系统当前 timezone/locale 格式化
   - 提供“日期时间”“仅日期”“图表小时轴”“图表天轴”几种展示模式
2. 所有 UI 展示统一改为调用该工具：
   - 概览页最近同步时间
   - 日志表创建时间
   - 趋势图 x 轴与 tooltip
3. 保持数据存储与接口返回不变，只修正展示层

## 取舍

### A. 前端统一格式化

优点：
- 改动最小
- 兼容当前混合数据源
- 天然使用 WebView 所在系统时区

缺点：
- 后端仍保留 epoch/ISO 混合语义

### B. 后端先归一化，再前端显示

优点：
- 响应语义更整齐

缺点：
- 会扩大改动面，并触及现有测试与存储约定

## 决策

采用 A。当前问题的根因在于前端没有统一展示层，而不是存储本身不可用。
